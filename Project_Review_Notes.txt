=======================================================
OMNICARE - PROJECT REVIEW CHEAT SHEET
=======================================================

1. WHAT IS OMNICARE?
-------------------------------------------------------
OmniCare is a comprehensive, all-in-one healthcare web application. Its goal is to bridge the gap between patients and healthcare services. Instead of using different apps to book a doctor, find a hospital, or request blood, OmniCare provides a centralized dashboard for patients to manage their entire health journey.

2. THE TECHNOLOGY STACK (MERN-like)
-------------------------------------------------------
* Frontend (The User Interface):
  - Next.js & React: Used to build a fast, single-page application experience.
  - Tailwind CSS: Used for modern, responsive, and clean styling.
  - Lucide React: Used for the beautiful, consistent icons throughout the app.

* Backend (The Server/API):
  - Node.js & Express.js: A lightweight, robust Javascript backend that handles API requests from the frontend.

* Database:
  - MongoDB (with Mongoose): A NoSQL database hosted on MongoDB Atlas. It stores hospitals, doctors, blood donors, appointments, and user profiles.

3. CORE FEATURES IMPLEMENTED (The 80%)
-------------------------------------------------------
A. Live Hospital Tracking:
   - What it does: Shows a list of nearby hospitals and their real-time occupancy load (e.g., 82% full - "High Load" / 20% full - "Available").

B. Doctor Search & Booking:
   - What it does: Allows patients to search for doctors by name or filter by specialty (e.g., Cardiologist, Pediatrician).
   - Interaction: Users can click "Book Now" to open an interactive modal, pick a date and time, and confirm the booking.

C. Blood Donor Network:
   - What it does: Connects patients in need of blood with local donors. Patients can filter donors by blood type.
   - Interaction: Click "Send Request" to instantly ping a donor, or click "Register as Donor" to open a form for new users to volunteer.

D. My Appointments:
   - What it does: A clean dashboard showing all the appointments a patient has booked, tracking their status (Confirmed, Pending, Cancelled).

E. Patient Profile & Medical Records:
   - What it does: A hub for the patient's data. It shows quick vitals (Heart rate, Blood Pressure) and the number of active appointments.
   - Interaction: Users can update their Name, Email, and Blood Group (which saves to the backend), and view their Medical Records.

4. HOW IT WORKS (The Architecture)
-------------------------------------------------------
1. The UI (Client): When a user clicks something (e.g., "Search Doctors"), React uses `fetch()` to send an HTTP GET request.
2. The API (Server): The Express server receives the request at the endpoint (e.g., `GET /api/doctors`).
3. The Database: Mongoose queries the MongoDB database, grabs the data, and sends it back as JSON.
4. The Result: React takes that JSON data and instantly updates the screen without reloading the webpage.


=======================================================
QUESTIONS THEY MIGHT ASK & HOW TO ANSWER
=======================================================

QUESTION: "What is currently missing from this project?" OR "What is the remaining 20% to be implemented?"

ANSWER: 
"If we had more time to finish the remaining 20%, we would implement the following three features:

1. Authentication & Security (Login/Signup): 
   Currently, the app assumes a mock user is logged in. The next immediate step is building secure user registration and login using JSON Web Tokens (JWT) and hashing passwords with bcrypt.

2. Provider / Doctor Dashboard: 
   Right now, we only have the 'Patient' view. We need to build a dashboard for Doctors and Admins so they can log in, view their schedule, and accept or reject the appointment requests coming from patients.

3. Live Geolocation & WebSockets: 
   Instead of using static location data, we plan to integrate the Google Maps API to use the user's actual GPS coordinates. We also want to upgrade the hospital 'refresh' buttons to use WebSockets (Socket.io) so the hospital load metrics update 100% live on the screen without clicking refresh."
